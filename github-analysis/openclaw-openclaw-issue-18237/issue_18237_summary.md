這個 GitHub issue 描述了一個問題，即代理程式在執行長時間的程序（例如編碼代理程式或建構流程）時，無法有效率地接收結果。目前的阻塞、輪詢或外部解決方案都很耗資源、不可靠或脆弱。

**提出的解決方案** 是引入一個「非同步執行回呼」（async exec callback）模式。在這個模式下，`exec` 指令會以 `callback=true` 啟動一個程序。代理程式的回合會立即結束，當程序完成時，結果會自動以系統事件或工具結果的形式注入回相同的會話中，並喚醒代理程式來處理。

**優點：**
* 簡化工作流程。
* 無需子代理程式、輪詢或外部掛鉤。
* 使 OpenClaw 成為更有效的編碼代理程式協調器。
