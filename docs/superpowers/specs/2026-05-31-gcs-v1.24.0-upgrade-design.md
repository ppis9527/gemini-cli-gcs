# GCS v1.24.0 穩定性升級設計 #project-gemini-cli #module-gcs #2026-05-31

## 1. 概述 (Overview)
本計畫旨在將 GCS (Context Governance System) 從 v1.23.0 升級至 v1.24.0。核心目標是提升系統在背景壓縮與環境隔離方面的穩定性，並引入多階層觸發機制。

## 2. 核心變更 (Core Changes)

### 2.1 多階層背景壓縮 (Multi-Threshold Background Compact)
- **機制**：從單一 20% 閾值升級為 `20%, 30%, 40%, 50%` 四個階層。
- **追趕邏輯**：若 Token 快速增長跨越多個閾值，系統將按順序補齊所有遺漏的壓縮任務。
- **去重**：確保同一閾值在單次 Session 中僅觸發一次。

### 2.2 環境隔離與預檢 (Environment & Preflight)
- **私有 Venv**：強制在 `.gemini/gcs-venv/` 建立專案專屬的 Python 虛擬環境。
- **依賴管理**：透過 `requirements.txt` 定義 `tree-sitter` 及其解析器。
- **預檢腳本**：`gcs_preflight.py` 將在每次背景任務執行前檢查環境，避免靜默失敗。

### 2.3 智慧攔截器 (Intelligent Interceptor)
- **意圖掃描**：`gcs_intercept.js` 現在會掃描 Prompt 中提及的文件路徑。
- **自動重灌**：若偵測到即將編輯的文件處於 Skeleton 狀態，將自動觸發重灌流程，確保代碼編輯的精確度。

## 3. 升級流程 (Upgrade Workflow)

### 3.1 本地狀態保護
1. 提交 `cli/*.sh` 的個人化修改（BTC 價格顯示、路徑調整）。
2. 清理 `.gemini/` 下的舊版狀態文件（checkpoint, log, monitor_state）。

### 3.2 合併與更新
1. 執行 `git merge origin/chore/gcs-relocate-and-fixes`。
2. 處理可能產生的衝突（預期衝突極低）。

### 3.3 環境配置
1. 建立 `.gemini/gcs-venv`。
2. 安裝 `tree-sitter`, `tree-sitter-python`, `tree-sitter-javascript`, `tree-sitter-typescript`。

## 4. 驗證標準 (Validation)
1. **單元測試**：`node tests/test_token_monitor.js` 必須顯示 `ok`。
2. **環境測試**：`python3 src/gcs/gcs_preflight.py` 必須顯示 `GCS_PREFLIGHT_OK`。
3. **整合測試**：模擬 21% Token 使用率，驗證 `tmux` 通知與背景進程啟動。
