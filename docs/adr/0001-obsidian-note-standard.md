# ADR 0001: Obsidian 技術文件規範
**日期**: 2026-02-25
**狀態**: 已通過 (Accepted)

## 上下文 (Context)
為了確保在 Obsidian 中的技術知識易於檢索且美觀，我們需要統一的 Markdown 格式規範。

## 決策 (Decision)
1. 必須包含 YAML Frontmatter。
2. 核心內容使用 Callouts ([!info], [!tip], [!warning]) 區分。
3. 關鍵術語與相關技能使用 [[Wikilinks]]。

## 後果 (Consequences)
- 優點: 視覺層次分明，知識鏈結化。
- 缺點: 生成檔案時需額外處理格式。
